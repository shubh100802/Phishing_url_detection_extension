Access Guidance & Security

🔒
Credentials are never shown publicly. For local development only, an admin account is auto-seeded after DB connect. Always rotate or change seeded passwords immediately.

Admin Access (Restricted)

How to obtain: Use authorized credentials only. Request access from the maintainer. Never hardcode or publish secrets.
Rotation: Change the seeded admin password on first login. Prefer a password manager.
Production: Disable seeding via env flag and provision admin securely.

User Access

  1. Open /frontend/usersignup.html and create an account.
  2. Log in via the user login page. Token is stored in localStorage.
  3. Logout clears auth and remembered fields.

Best Practices

Do not share credentials Use .env for secrets Disable seeding in prod Rotate passwords regularly Least privilege

Troubleshooting

Backend up on PORT=3001 Frontend served at 127.0.0.1:5500 CORS allows localhost & 127.0.0.1 Clear localStorage to reset
